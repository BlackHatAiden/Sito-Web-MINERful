Examples

From description to specification

A description consists in exposing in natural language the reality to be modeled. Given a description you can proceed by creating a specification, which will have to respect and model in a logical and precise way the reality exposed by the description, minimizing ambiguities. It is represented through declare templates.


Description


  • Every process instance starts with receiving an invoice. --> Receive invoice


  • Whenever an invoice is recorded into the database, it was received beforehand. --> Record invoice in database


  • When an invoice is received, the current account will be checked at some point afterwards. No current account is checked, unless an invoice was received at some point before. --> Check current account


  • The account will be checked at least once in every process instance. --> Check current account(for every process instance 1..*)


  • Every single time an invoice is settled, the current account was checked before. --> Settle invoice

Specification


From specification to cases

Given a specification, then a logical modeling based on declare templates of a reality described above, you get the cases that are the possible combinations that may occur from the specifications of the model in question.

For example in the first image specification we see how each combination should start with c because of the template Init(c) and immediately after an a there must be at least one d because of the template Chain Response(d,a) and if there is a b then it must take at least one a not necessarily following because of the template Response(b,a).

In the second image specification we see how each combination should end with a because of the template End(a) and if there is a b then it must take at least one a not necessarily following but the combination must end with a because of the template Response(b,a) with cardinal of b is (1..1) and End(a) template, and it cannot coexist in a combination d and e and if the combination must end with a then it must be preceded by at least a c because of the template Chain Precedence(c,a) with End(a) .
For clarity visit declare templates.
